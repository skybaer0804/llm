# 01. Qwen3 모델 구성 전략

Mac M4 Pro 64GB 환경에서 최상의 성능과 효율을 내기 위한 Qwen3 기반 멀티 에이전트 모델 구성 전략을 설명합니다.

## 1. 에이전트별 모델 매핑

본 시스템은 '설계-구현-검증'의 3단계 협업 사이클을 위해 최적화된 3개의 에이전트를 운영합니다.

| 역할 | 에이전트 명칭 | Ollama 모델 태그 | 특징 및 용도 | 메모리 점유 |
| :--- | :--- | :--- | :--- | :--- |
| **🧠 설계자 (Architect)** | `Planner` | `qwen3-coder-next:q4_K_M` | 요구사항 분석, 전체 아키텍처 설계, 256K 컨텍스트 활용 | 약 52GB |
| **💻 개발자 (Coder)** | `Dev_Senior` | `qwen3-coder:32b` | 실제 기능 구현, 리팩터링, 고속 코드 생성 | 약 19GB |
| **👀 검수자 (Reviewer)** | `Dev_Reviewer` | `qwen3:14b` | 코드 리뷰, 테스트 케이스 생성, 버그 탐지 | 약 9GB |

## 2. 왜 Qwen3 시리즈인가?

1.  **코딩 특화 성능**: Qwen3-Coder 계열은 최신 벤치마크(SWE-bench 등)에서 최고 수준의 코딩 능력을 보여줍니다.
2.  **일관된 프롬프트 스타일**: 동일한 모델 패밀리를 사용함으로써 에이전트 간 대화의 일관성을 높이고 프롬프트 엔지니어링 효율을 극대화합니다.
3.  **효율적인 메모리 사용**: 80B MoE(Mixture of Experts) 구조의 Coder-Next부터 경량화된 14B 모델까지 라인업이 다양하여 64GB 시스템 내에서 역할 분담이 용이합니다.

## 3. 하드웨어 제약 사항 및 대응

- **단일 인스턴스 제한**: `qwen3-coder-next:q4_K_M`은 약 52GB를 점유하므로, OS 및 기타 프로세스 여유 공간을 고려할 때 **단독으로만 구동**이 가능합니다.
- **병렬 구동**: 개발자(32b, 19GB)와 검수자(14b, 9GB) 모델은 합계 약 28GB로, 두 모델을 동시에 메모리에 올려 빠른 피드백 루프를 구성할 수 있습니다.
- **순차 라우팅**: 설계자가 작업을 마친 후 메모리를 반납하면 개발자와 검수자가 바통을 이어받는 **순차 실행(Sequential Execution)** 구조를 채택합니다.
