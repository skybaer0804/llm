# 02. 시스템 사양 및 요구사항

성공적인 로컬 LLM 및 AutoGen 시스템 구축을 위한 최적의 하드웨어 사양과 환경 요구사항을 안내합니다.

## 1. 하드웨어 사양 (Dual Mac 분산 아키텍처)

본 시스템은 맥북과 맥미니의 자원을 썬더볼트로 결합한 분산 환경에 최적화되어 있습니다.

### A. 메인 워커 (Mac Mini M4 Pro)
- **칩셋**: Apple M4 Pro (14코어 CPU, 20코어 GPU)
- **메모리**: **64GB 통합 메모리 (Unified Memory)**
- **역할**: **Worker 전담** (51GB 거대 모델 `qwen3-coder-next` 실행)
- **네트워크**: 썬더볼트 브리지 (고속 데이터 전송 전용)

### B. 지능형 관문 (MacBook Pro M4 Pro)
- **칩셋**: Apple M4 Pro
- **메모리**: **24GB 통합 메모리 (Unified Memory)**
- **역할**: **Gateway 전담** (8~10GB급 라우터 모델 `llama3.1:8b` 실행) 및 개발 UI
- **네트워크**: 썬더볼트 브리지 (169.254.x.x 대역 활용)

## 2. 모델 실행 및 분산 능력

분산 아키텍처를 통해 단일 기기의 메모리 한계를 극복합니다.

- **역할 분리 (Decoupling)**: 분석/보안은 맥북(Gateway)이, 실제 연산/코딩은 맥미니(Worker)가 전담하여 리소스 간섭 최소화.
- **썬더볼트 초고속 통신**: 40Gbps 이상의 대역폭을 가진 썬더볼트 브리지를 통해 원격 모델 호출 시 지연 시간(Latency)을 최소화.
- **메모리 최적화**: 맥북의 24GB 메모리는 Cursor 및 UI 도구에, 맥미니의 64GB는 오직 대형 LLM에 집중 할당.

## 3. 소프트웨어 요구사항

시스템 구축을 위해 다음 도구들이 설치되어야 합니다.

- **OS**: macOS 최신 버전 (M4 Pro 성능 최적화 반영)
- **런타임**: Python 3.10 이상, Node.js (OpenWebUI, Claude Code CLI 등 활용 시)
- **버전 관리**: Git (Claude Code CLI 연동 및 코드 히스토리 관리 필수)
- **가상화**: Docker Desktop for Mac
- **LLM 엔진**: Ollama (각 기기별 설치 필수)
- **프레임워크**: AutoGen 0.4.x 이상

## 4. 핵심 환경 설정 전략

- **TDD (Test Driven Development)**: 개발 에이전트가 코드를 작성할 때 항상 테스트 코드를 동반하도록 구성.
- **분산 모델 라우팅**: `router.py`를 통해 맥북(Gateway)과 맥미니(Worker) 간의 트래픽 자동 분산.
- **컨텍스트 공유**: 썬더볼트 브리지 네트워킹을 통한 기기간 빠른 데이터 동기화 및 API 호출.
