## 맥미니에서 Python 가상환경 생성 및 활성화

맥미니에서 AutoGen과 같은 새로운 프로젝트를 시작하실 때 가상 환경을 사용하는 것은 매우 현명한 선택입니다. 단순히 '권장 사항'을 넘어, 시스템의 안정성을 지키는 '보호막' 역할을 하기 때문입니다.
가상 환경(Virtual Environment)에 설치하는 것이 안전한 구체적인 이유는 다음과 같습니다.
1. 의존성 충돌 방지 (Dependency Hell 예방)
Python 프로젝트마다 필요한 라이브러리의 버전이 다를 수 있습니다.
상황: 프로젝트 A는 Library-X의 버전 1.0이 필요하고, AutoGen은 버전 2.0이 필요할 수 있습니다.
위험: 가상 환경 없이 시스템 전체(Global)에 설치하면, 한쪽 프로젝트는 반드시 작동하지 않거나 오류를 일으키게 됩니다. 가상 환경은 각 프로젝트를 독립된 방에 가두어 서로 간섭하지 않게 합니다.
2. 시스템 Python 환경 보호
macOS(맥미니)에는 운영체제 자체의 기능을 수행하기 위해 기본적으로 설치된 Python이 있습니다.
위험: 시스템 Python 환경에 무분별하게 라이브러리를 설치하거나 수정하면, macOS의 시스템 도구나 기본 앱들이 오작동할 위험이 있습니다. 가상 환경은 OS가 사용하는 영역을 건드리지 않고 사용자의 작업 영역만 분리해 줍니다.
3. 깔끔한 실험과 삭제
AutoGen은 대규모 언어 모델(LLM) 프레임워크로, 설치 시 수많은 관련 라이브러리들이 함께 설치됩니다. 
장점: 만약 AutoGen을 테스트해 본 뒤 더 이상 필요 없게 되었을 때, 가상 환경 폴더만 지우면 시스템이 다시 깨끗해집니다. 시스템 전체에 설치했다면 어떤 라이브러리가 AutoGen 때문에 설치된 것인지 일일이 찾아 지우기 매우 어렵습니다.
4. 재현성 (Reproducibility) 확보
나중에 다른 컴퓨터로 옮기거나 팀원과 공유할 때 유리합니다.
장점: 가상 환경 내의 패키지 목록을 추출(pip freeze > requirements.txt)하면, 다른 환경에서도 내가 작업했던 것과 정확히 동일한 버전의 환경을 즉시 구축할 수 있습니다.
