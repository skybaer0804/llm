# 터미널 기본 명령어 문법

## cat (Concatenate)
원래는 파일 내용을 화면에 출력할 때 쓰는 명령어입니다. 하지만 여기서는 뒤에 오는 기호들과 조합되어 "입력받은 텍스트를 처리하라"는 역할을 합니다.

## > (Redirection)

화면에 출력될 내용을 파일로 돌리는(저장하는) 기호입니다.
`cat > router.py`: "내용을 보여주지 말고 router.py라는 파일 안에 차곡차곡 쌓아라"라는 의미입니다. (기존에 파일이 있다면 덮어씁니다.)

## << 'EOF' (Here Document)

이게 핵심입니다! "어디서부터 어디까지가 파일 내용인지" 컴퓨터에게 알려주는 약속입니다.
`<<`: "이제부터 긴 문장을 입력할 건데, 특정 단어가 나오면 입력을 끝낼게"라는 뜻입니다.
`'EOF'`: End Of File의 약자로, 입력의 끝을 알리는 '신호탄'입니다. (꼭 EOF가 아니어도 되지만 관례상 가장 많이 씁니다.)

### 요약하자면 이 과정이 일어납니다:
1.  **시작**: 터미널에 저 명령어를 치면 다음 줄로 넘어갑니다.
2.  **입력**: 준비하신 파일 전체 코드를 복사해서 붙여넣습니다.
3.  **종료**: 마지막 줄에 아무 글자 없이 `EOF`라고만 입력하고 엔터를 칩니다.
4.  **결과**: 방금 붙여넣은 모든 내용이 파일로 저장됩니다.

주로 메모장(`vi`, `nano`)을 켜기 귀찮거나, 스크립트로 파일을 한 번에 만들 때 사용하는 아주 유용한 테크닉입니다.
