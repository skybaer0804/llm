## 1. AutoGen 선택 + Qwen3-Coder-Next 메인으로 간다는 방향성

이 부분은 그대로 동의해도 좋습니다.

### AutoGen 쪽

- **멀티 에이전트 대화 흐름을 직접 설계 가능**
설계자 → 개발자 → 검수자 식의 반복 협업(Iteration)을 짜기 좋고,
메시지 라우팅 / 역할 분리도 명확해서 설계 의도가 잘 반영됩니다.
- **코드 실행 + 디버깅 루프**
AutoGen은 코드 실행/파일 시스템 연동이 잘 되어 있어
“작성 → 실행 → 에러 기반 수정” 구조의 **self-debug 루프**를 만들기에 적합합니다.
이건 말씀하신 대로 검수/테스트 에이전트에 핵심 기능입니다.


### Qwen3-Coder-Next를 메인으로 쓴다는 점

- Qwen3-Coder-Next는
    - 80B MoE (실제 활성 파라미터 3B 수준) 구조,
    - 256K 컨텍스트,
    - agentic coding + tool use + 환경 상호작용에 특화된 학습을 한 모델이라
“에이전트형 코딩 LLM”으로 쓰기에 굉장히 잘 맞는 선택입니다.[^2_1][^2_2]

이 정도 큰 그림은 아주 좋은 판단입니다.
