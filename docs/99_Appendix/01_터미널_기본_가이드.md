# 01. 터미널 기본 가이드

로컬 LLM 및 AutoGen 환경 운영을 위해 필수적인 터미널 명령어와 파일 관리 기법을 정리합니다.

## 1. 핵심 명령어 문법

### 파일 생성 및 텍스트 주입 (cat & Redirection)
메모장을 열지 않고 터미널에서 즉시 파일을 만들거나 내용을 채울 때 유용합니다.

- **`cat > file.py`**: 입력한 내용을 `file.py`로 저장합니다. (기존 내용 덮어쓰기)
- **`cat >> file.py`**: 기존 내용 뒤에 새로운 내용을 추가합니다.
- **`<< 'EOF' (Heredoc)`**: 특정 단어(EOF)가 나올 때까지의 모든 텍스트를 하나의 블록으로 처리합니다.

**사용 예시**:
```bash
cat > test.py << 'EOF'
print("Hello, AutoGen!")
EOF
```

## 2. 파일 편집 및 관리

- **파일 이동/이름 변경**: `mv old_name.py new_name.py`
- **간단한 편집 (nano)**: `nano router.py`
    - 저장: `Ctrl + O` → `Enter`
    - 종료: `Ctrl + X`
- **VS Code로 편집**: `code router.py` (VS Code 설치 시 가장 권장되는 방법)

## 3. 유용한 명령어 모음 (Cheat Sheet)

### Ollama & 모델 관리
- `ollama list`: 설치된 모델 목록 확인
- `ollama pull [model]`: 모델 다운로드
- `ollama rm [model]`: 모델 삭제
- `brew services restart ollama`: Ollama 서비스 재시작

### Docker 관리
- `docker ps`: 실행 중인 컨테이너 확인
- `docker logs -f [container]`: 실시간 로그 확인
- `docker restart [container]`: 컨테이너 재시작
- `docker system prune`: 미사용 이미지/볼륨 정리 (용량 확보)

### 시스템 모니터링
- `memory_pressure`: macOS 메모리 압박 상태 확인
- `top -u`: CPU 및 메모리 점유 프로세스 확인
- `df -h`: 디스크 여유 공간 확인
- `powermetrics --samplers gpu_power -n 1`: Apple Silicon GPU 전력/부하 확인

### GitHub CLI
- `gh issue list --state=open`: 열린 이슈 목록
- `gh issue comment [number] -b "message"`: 이슈에 댓글 추가
- `gh auth status`: 현재 로그인 상태 및 권한 확인
