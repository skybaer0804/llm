# Claude Code 실전 사용 팁 (by bcherny)

Claude Code 창시자(bcherny)가 공개한 생산성 향상을 위한 실전 팁 모음입니다.

## 1. 병렬 작업의 극대화
- **Git Worktree 활용**: 3~5개의 git worktree를 동시 실행하고 각각 별도의 Claude 세션을 운영하는 것이 최고의 생산성 팁입니다.
- **Shell Alias 설정**: worktree 간 빠른 전환을 위해 `za`, `zb`, `zc` 등의 alias를 사용하세요.
- **전용 Worktree**: 로그 확인 및 BigQuery 실행 전용 "analysis" worktree를 별도로 운영하기도 합니다.

## 2. 복잡한 작업은 Plan Mode로 시작
- **선 계획 후 구현**: 복잡한 작업은 반드시 `plan` 모드로 시작하여 구조적 결함을 먼저 파악하세요.
- **이중 검토**: 한 Claude가 계획을 짜면, 다른 Claude 세션을 띄워 스태프 엔지니어 역할로 검토시키세요.
- **즉시 재계획**: 작업이 꼬이면 무리하게 밀어붙이지 말고 즉시 `plan` 모드로 돌아가 재설계하세요.

## 3. CLAUDE.md의 지속적 업데이트
- **실수 반복 방지**: 매 수정 시 "CLAUDE.md를 업데이트해서 같은 실수를 반복하지 않게 해라"라고 지시하세요.
- **규칙의 자율 문서화**: Claude는 스스로 지켜야 할 규칙을 문서화하는 데 매우 능숙합니다.
- **실수율 측정**: CLAUDE.md를 지속 수정하며 실수율이 줄어들 때까지 반복 투자하세요.

## 4. 자체 Skill 및 슬래시 커맨드 생성
- **반복 작업 자동화**: 하루에 한 번 이상 반복하는 작업은 Skill이나 슬래시 커맨드로 만드세요.
- **예시**: `/techdebt` (중복 코드 제거), 컨텍스트 덤프 동기화, 분석 에이전트 등.

## 5. 버그 수정의 위임
- **컨텍스트 전달**: Slack MCP나 CI 테스트 로그, Docker 로그를 Claude에게 직접 전달하고 "fix"라고만 지시하세요.
- **결과 중심 지시**: 구체적인 방법 대신 "Go fix the failing CI tests"와 같이 목표 중심으로 지시하세요.

## 6. 프롬프트 수준 고도화
- **엄격한 리뷰 요청**: "테스트 통과 전까지 PR 만들지 마"와 같이 엄격한 기준을 부여하세요.
- **차이점 증명**: main과 feature 브랜치 간의 동작 차이를 diff하도록 요청하세요.
- **우아한 솔루션**: 결과가 평범하면 "지금까지의 배움을 바탕으로 이걸 버리고 더 우아한(Elegant) 솔루션을 구현해"라고 지시하세요.

## 7. 터미널 및 환경 최적화
- **Ghostty 터미널**: 동기화 렌더링과 24비트 컬러를 지원하는 Ghostty를 선호합니다.
- **상태바 커스터마이징**: `/statusline`으로 컨텍스트 사용량과 git 브랜치를 상시 표시하세요.
- **음성 받아쓰기**: 타이핑보다 3배 빠른 음성 입력을 적극 활용하세요.

## 8. 서브에이전트 활용
- **부하 분산**: 요청 시 `use subagents`를 추가하여 메인 에이전트의 컨텍스트를 깨끗하게 유지하세요.
- **라우팅 Hook**: 권한 요청이나 고난도 추론은 Opus 4.5 등으로 자동 라우팅되도록 설정하세요.

## 9. 데이터 분석 자동화
- **CLI 연동**: Claude Code에서 직접 BigQuery 등 데이터베이스 CLI를 사용하여 분석을 수행하세요.
- **Skill 체크인**: 분석용 Skill을 git에 커밋하여 팀 전체가 공유하세요.

## 10. Claude를 통한 지속적 학습
- **스타일 변경**: `/config`에서 `Explanatory`나 `Learning` 스타일을 활성화하여 'Why'를 이해하세요.
- **시각화 요청**: 복잡한 코드는 HTML 슬라이드나 ASCII 다이어그램으로 그리도록 요청하세요.
- **간격 반복 학습**: 사용자가 이해한 내용을 Claude가 테스트하는 skill을 구축하세요.
