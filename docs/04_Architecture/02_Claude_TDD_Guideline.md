# 02. Claude TDD 가이드라인

이 문서는 맥북에서 작업을 수행하는 Claude Code(또는 Cursor AI)가 반드시 준수해야 할 TDD(테스트 주도 개발) 원칙을 정의합니다.

## 1. TDD 핵심 원칙: "No Test, No Code"

모든 구현 작업은 반드시 다음 단계를 거쳐야 합니다.

1.  **Red (테스트 작성)**: `tasks/pending/Task-XX.md`에 정의된 설계와 테스트 시나리오를 바탕으로 테스트 코드를 먼저 작성합니다.
2.  **Fail (실패 확인)**: 작성한 테스트를 실행하여 실패하는지 확인합니다. (기능 구현 전이므로 당연히 실패해야 함)
3.  **Green (기능 구현)**: 테스트를 통과하기 위한 최소한의 코드를 작성합니다.
4.  **Refactor (리팩토링)**: 테스트 통과를 유지하면서 코드를 최적화합니다.

## 2. Claude를 위한 지시사항

- **설계서 우선**: 작업을 시작하기 전 반드시 `tasks/pending/` 폴더의 최신 지시서를 정독하십시오.
- **테스트 코드 위치**: 프로젝트의 관례에 따라 `tests/` 폴더 아래에 관련 테스트 파일을 생성하십시오.
- **커밋 메시지**: TDD 단계를 명확히 알 수 있도록 커밋 메시지를 작성하십시오. (예: `test: add login failure cases`, `feat: implement login logic`)
- **검수 준비**: 모든 테스트가 로컬에서 통과된 것을 확인한 후에만 Push를 진행하십시오.

## 3. 검수자와의 협업

- 검수자(Qwen3)는 설계서에 명시된 테스트 시나리오를 기준으로 코드를 검증합니다.
- 만약 검수자가 재지시를 내린다면, 이는 테스트 케이스가 누락되었거나 설계서의 요구사항을 충족하지 못했음을 의미합니다.
- **주의**: 검수자가 잘못된 테스트를 요구한다고 판단될 경우, 즉시 설계서(`Task-XX.md`)와 대조하여 논리적 근거를 제시하십시오.

## 4. 금지 사항

- 테스트 코드 없이 기능 코드만 작성하여 Push하는 행위.
- 설계서에 없는 기능을 임의로 추가하는 행위.
- 테스트가 실패하는 상태로 Push하는 행위.
