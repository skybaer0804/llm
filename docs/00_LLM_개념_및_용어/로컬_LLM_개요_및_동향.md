# 로컬 LLM 개요 및 동향 (2026.02.09)

## 1. LLM (Large Language Model, 대규모 언어 모델) 개요
- **분류**: LLM은 로컬(Local)과 프론티어(Frontier) 분야로 나뉩니다.
- **로컬 LLM**: 개인 PC, 사내 서버, 안전한 클라우드에 직접 구축한 모델.
- **프론티어 LLM**: OpenAI(ChatGPT), Google(Gemini) 등 기업이 제공하는 대형 모델. 성능과 스펙이 매우 우수함.
- **로컬 LLM의 특징**: 공개된 모델을 다운로드하여 직접 실행 및 튜닝 가능.

## 2. 로컬 LLM 주요 모델
1.  **DeepSeek** (중국, 알고리즘 중심)
2.  **Qwen** (중국, 알리바바)
3.  **Gemma** (미국, 구글)
4.  **Llama** (미국, 메타)
- 최근 마이너하지만 성능 좋은 오픈 소스 모델들이 많이 공개되는 추세입니다. 목적에 맞는 꾸준한 업데이트가 이루어지는 모델을 선택하는 것이 중요합니다.

## 3. LLM 설치 및 환경 구축
- **경량 모델**: 사양이 낮은 PC에서도 간단한 작업용으로 사용 가능.
- **Mac Mini 환경**: `node.js`, `Homebrew`, `nvm`을 설치하고 `Ollama`를 통해 다양한 모델 구동 가능.
- **주요 과제**:
    - 다중 에이전트(설계자, 개발자, 리뷰어) 구성 시 `AutoGen` 프레임워크 활용 검토 (설치 및 설정 과정의 복잡성 해결 필요).
    - 대형 모델 실행 시 메모리 부족 문제 해결 방안 필요.
    - 고속 데이터 전송을 위해 **썬더볼트 5 케이블**(80Gbps) 활용 권장.

## 4. LLM 상용 서비스화 고려 사항
- **가이드레일(Guardrails) 적용**: 부적절한 질문 차단 및 민감 정보 유출 방지 필수.

## 5. 자동화 및 에이전트 활용 (오픈클로/몰트봇)
- 보안 이슈 해결 방안 모색 중.
- iMessage, Discord, Telegram 등을 통해 외부에서도 로컬 LLM과 통신 가능.
- 24시간 구동 및 스케줄링을 통해 특정 작업 자동화 가능.
